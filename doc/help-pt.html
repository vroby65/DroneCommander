Drone Commander

Drone Commander é uma aplicação web interativa que permite simular o controlo de voo de drones usando programação visual. Você cria um programa de voo arrastando e soltando blocos Blockly e, em seguida, observa o drone executar esses comandos num ambiente 3D em tempo real alimentado por Three.js.

Não é necessária experiência em programação — basta encaixar os blocos e pressionar Executar!

Início rápido: Arraste um bloco Start, conecte um bloco decolar, depois um bloco aterrar e clique em ▶ Executar. O seu drone vai descolar e aterrar!

Visão Geral da Interface

A aplicação está dividida em dois painéis principais separados por um divisor redimensionável:

  • Painel Esquerdo — O editor Blockly onde você constrói o seu programa de voo.
  • Painel Direito — A vista de simulação 3D com uma barra de estado na parte inferior.

Barra de menu do painel esquerdo (topo do editor):

  • Seletor de idioma — Altera o idioma da interface.
  • Novo — Cria um novo programa vazio.
  • Carregar — Carrega um programa Blockly de um ficheiro .xml no seu computador.
  • Salvar — Guarda o seu programa atual como um ficheiro .xml.
  • Menu de cenário — Escolhe um ambiente 3D (veja Cenários).
  • Menu gráfico — Escolhe uma predefinição de qualidade gráfica (veja Perfis Gráficos).

Barra de ferramentas do painel direito (acima da vista 3D):

  • 🗖 / 🗗 — Alterna a vista 3D para ecrã inteiro (oculta o editor e o painel de estado).
  • Nome do ficheiro — O nome do programa atual.
  • ▶ Executar — Executa o programa Blockly.
  • ■ Parar — Para a simulação; clique novamente para redefinir o drone.

O Editor Blockly

O editor Blockly é um ambiente de programação visual. Você cria um programa arrastando blocos da caixa de ferramentas (a barra lateral à esquerda do editor) para a área de trabalho (a grande área branca) e encaixando-os como peças de puzzle.

Como construir um programa

  1. Cada programa deve começar com um bloco Start (da categoria Flow).
  2. Conecte blocos de comando de Drone abaixo do bloco Start na ordem em que deseja que sejam executados.
  3. Opcionalmente, termine com um bloco End.
  4. Clique em ▶ Executar para ver o drone executar o seu programa.

Ações de edição comuns

  • Arrastar — Move um bloco da caixa de ferramentas ou pela área de trabalho.
  • Clique direito num bloco para opções de contexto: Duplicar, Eliminar, Adicionar Comentário, Desativar, Entradas em Linha, etc.
  • Entradas numéricas — Clique num campo numérico e digite um valor. Também pode conectar um bloco de Matemática (ex.: um bloco de número ou aritmético) às ranhuras de entrada.
  • Entradas booleanas (ex.: o bloco fumaça) aceitam blocos Verdadeiro/Falso, blocos lógicos ou blocos de sensores.
  • Desativar um bloco — Clique direito > Desativar Bloco para removê-lo temporariamente do programa sem o eliminar.
  • Colapsar — Clique direito > Colapsar Bloco para reduzir a desordem visual.

Referência de Blocos

A caixa de ferramentas no lado esquerdo do editor está organizada nas seguintes categorias:

Lógica Laços Matemática Texto Listas Variáveis Funções Sensores Drone Flow

Lógica

Condições if/else padrão, operadores lógicos (e, ou, não), valores booleanos (verdadeiro/falso), operadores de comparação (=, ≠, <, ≤, >, ≥) e o operador ternário.

Laços

Blocos de repetição por um número fixo de vezes, laços while/until, laços de contagem (for), laços for-each e instruções de controlo de laço (break, continue).

Matemática

Constantes numéricas, operações aritméticas (+, −, ×, ÷, ^), raiz quadrada, valor absoluto, funções trigonométricas, arredondamento, números aleatórios e verificações de propriedades numéricas (par, ímpar, primo, etc.).

Texto

Constantes de string, concatenação de texto, comprimento, substring, pesquisa de caracteres, conversão de maiúsculas/minúsculas, remoção de espaços e impressão na consola.

Listas

Criar listas, obter/definir itens, encontrar sublistas, ordenar, inverter, verificar se está vazia e dividir/juntar strings.

Variáveis

Crie e use variáveis para armazenar e recuperar valores durante a execução do programa.

Funções

Defina procedimentos reutilizáveis (com ou sem valores de retorno) para organizar o seu código.

Blocos de Drone

Estes são os blocos principais para controlar o drone. Os blocos marcados como requer voo só terão efeito enquanto o drone estiver no ar; são silenciosamente ignorados se o drone estiver no solo.

Bloco Descrição Requer Voo
decolar Liga as hélices do drone e eleva-o a uma altura de 10 metros. O drone entra em estado de voo. Não
aterrar Desce o drone até ao solo, para as hélices e sai do estado de voo. O drone aterra na altura do terreno abaixo dele. Sim
definir altitude [N] Altera a altitude do drone para o valor especificado (em metros). O drone faz uma transição suave para a altitude alvo. Sim
alterar altitude [N] Ajusta a altitude do drone por um valor relativo (ex.: +5 para subir 5m, -3 para descer 3m). Sim
definir ângulo [N] Roda o drone para uma direção específica (0–359 graus, onde 0 = norte). Sim
alterar ângulo [N] Roda o drone por um número relativo de graus (ex.: 90 para virar à direita, -90 para virar à esquerda). Sim
deslizar [N] Move o drone lateralmente (esquerda/direita) pela distância especificada. Valores positivos movem para a direita, negativos para a esquerda. Sim
andar [N] Move o drone para a frente pela distância especificada (em metros). O drone move-se na direção para a qual está virado. Sim
andar subindo avançar [N] subir [N] Move o drone para a frente enquanto altera a altitude. avançar é a distância em metros, e subir é a alteração relativa de altitude; valores negativos descem até à altitude segura. Sim
ir para x [X] y [Y] z [Z] Move o drone para uma posição absoluta, onde Y é a altitude. O drone roda e inclina-se para o destino usando a velocidade atual. Sim
mover por x [X] y [Y] z [Z] Move o drone por um deslocamento relativo a partir da posição atual. O drone roda e inclina-se para o destino usando a velocidade atual. Sim
curva abs x [X] y [Y] z [Z] xd [XD] yd [YD] zd [ZD] Faz o drone voar numa curva que passa por três pontos: a posição atual, o ponto intermédio absoluto X/Y/Z e o destino absoluto XD/YD/ZD. A velocidade atual controla o movimento. Sim
curva x [X] y [Y] z [Z] xd [XD] yd [YD] zd [ZD] Faz o drone voar numa curva que passa por três pontos com coordenadas relativas: a posição atual, o ponto intermédio com deslocamento X/Y/Z a partir da posição atual e o destino com deslocamento XD/YD/ZD a partir desse ponto intermédio. A velocidade atual controla o movimento. Sim
voltar à base Leva o drone de volta à base em X=0, Z=0 e altitude 10. O drone roda e inclina-se para a base usando a velocidade atual. Sim
esperar [N] Pausa o programa pelo número especificado de segundos antes de continuar para o próximo bloco. Não
fumaça [Verdadeiro/Falso] Ativa ou desativa o efeito de rasto de fumo. Quando ativado, o drone deixa um rasto visível enquanto voa. Não
velocidade [N] Define a velocidade de movimento do drone (0–10). Valores mais altos fazem o drone mover-se mais rápido durante comandos andar, andar subindo, ir para, mover por, curva abs, curva e voltar à base. A velocidade predefinida é 1.0. Não
Nota: O drone só pode executar andar, andar subindo, deslizar, ir para, mover por, curva abs, curva e voltar à base enquanto voa. definir_ângulo e alterar_ângulo mudam a direção do drone sem o mover.

Blocos de Sensores

Os blocos de sensores retornam dados em tempo real da simulação do drone. Podem ser ligados a entradas numéricas ou booleanas de outros blocos (ex.: dentro de condições de blocos if/while).

Bloco Retorna Descrição
tecla [TECLA] pressionada? Booleano Retorna verdadeiro se a tecla especificada estiver atualmente pressionada. Teclas disponíveis: Seta para Cima/Baixo/Esquerda/Direita, Espaço, Enter, A–Z.
X Número Retorna a coordenada X atual do drone (posição este-oeste).
Z Número Retorna a coordenada Z atual do drone (posição norte-sul).
Altitude Número Retorna a altitude atual do drone em metros.
Direção Número Retorna a direção atual do drone em graus (0–359).
velocidade Número Retorna a configuração atual de velocidade do drone (0–10).

Usar sensores — padrões sugeridos

  • Condução por teclado: Use tecla pressionada dentro de um laço while verdadeiro com blocos se para fazer o drone reagir às setas do teclado em tempo real.
  • Altitude condicional: Use o sensor Altitude com uma comparação lógica para manter o drone acima de uma certa altura.
  • Aterragem automática: Verifique se a altitude está perto de 0 e então acione um bloco aterrar.

Blocos de Controlo de Fluxo

Bloco Descrição
Start O ponto de entrada de cada programa. Todos os comandos do drone devem ser conectados abaixo deste bloco. Tem uma forma de "chapéu" e não tem conexão na parte superior.
End Um bloco opcional para marcar o fim do programa. Quaisquer blocos abaixo dele não serão executados.
Dica: Se não incluir um bloco Start, o programa ainda será executado, mas pode executar blocos numa ordem inesperada. Comece sempre com Start.

Controlos de Simulação

Executar (▶)

Clique no botão verde ▶ na barra de ferramentas do painel direito para executar o seu programa Blockly. O programa é convertido para JavaScript e executado passo a passo. Durante a execução, o botão Parar fica ativo e o botão Executar fica desativado.

Parar (■)

Clique no botão vermelho ■ para parar o programa em execução. O drone para imediatamente. Clique em Parar novamente para redefinir o drone para a sua posição inicial.

O que acontece durante a execução

  • Cada bloco é executado sequencialmente com um pequeno atraso entre comandos para que possa ver o movimento.
  • O som do drone (ruído das hélices) é reproduzido durante o voo.
  • O painel de estado é atualizado em tempo real com posição, altitude e direção.
  • As hélices giram visivelmente durante o voo.
  • Os rastos de fumo são renderizados se o modo de fumo estiver ativado.

Vista 3D e Controlos de Câmara

A vista 3D mostra o drone a voar no cenário selecionado. Pode controlar a câmara livremente:

Ação Controlo
Orbitar (rodar vista) Clique direito + arrastar para rodar a câmara à volta do drone.
Repor vista Clique do meio (botão do scroll do rato) para posicionar a câmara atrás do drone.
Zoom Scroll do rato para ampliar e reduzir.
Primeira pessoa Clique esquerdo + arrastar enquanto o drone está a voar para mudar para uma vista de câmara de perseguição que segue o drone de perto.

Guias visuais do drone

Durante o voo, duas ajudas visuais aparecem (nos modos Equilibrado e Desempenho):

  • Marcador no solo — Um anel azul no solo diretamente abaixo do drone.
  • Linha de altitude — Uma linha vertical azul a ligar o drone ao seu marcador no solo, mostrando a altura.

Estas guias podem ser ocultadas mudando para o modo Qualidade ou definindo um perfil personalizado.

Painel de Estado

O painel de estado é mostrado na parte inferior do painel direito. Exibe a telemetria atual do drone e permite ajustes manuais:

Campo Descrição
X A posição este-oeste do drone. Editável — altere o valor e prima Enter para teletransportar o drone (com verificação de colisão).
Z A posição norte-sul do drone. Editável como X.
Altitude A altura do drone acima do solo. Editável — útil para posicionar rapidamente o drone.
Direção A direção do drone em graus (0–359). Editável — altere para rodar o drone instantaneamente.
Estado Mostra "No Solo" ou "Em Voo" dependendo do estado do drone.

Gestão de Ficheiros

Novo

Clique em Novo para limpar o programa atual e começar com uma área de trabalho nova contendo apenas um bloco Start. Será perguntado se deseja salvar o programa atual primeiro.

Salvar

Clique em Salvar para descarregar o seu programa atual como um ficheiro .xml. Será solicitado um nome para o ficheiro. O programa também é automaticamente salvo no armazenamento local do seu navegador.

Carregar

Clique em Carregar para abrir um ficheiro .xml do seu computador. O programa carregado substitui o conteúdo atual da área de trabalho.

Salvamento Automático

O seu programa é automaticamente salvo no armazenamento local do navegador sempre que faz uma alteração. Quando voltar à página ou a atualizar, o último programa salvo é restaurado automaticamente. O nome do ficheiro atual também é lembrado entre sessões.

Cenários

O Drone Commander inclui quatro ambientes 3D que pode alternar usando o menu no painel esquerdo.

Cenário Descrição
Campo de voo (Flight Field) Um campo de aviação relvado com árvores, bancos e colinas suaves. O ambiente predefinido.
Pista urbana (Urban Track) Um ambiente urbano com edifícios e cenário citadino.
Metrópole (Metropolis) Uma grande cidade com arranha-céus e uma disposição urbana densa.
Ilha tropical (Tropical Island) Uma ilha tropical com água, areia e vegetação insular.

Quando muda de cenário, o drone é redefinido para o centro do novo ambiente e a câmara volta à sua posição predefinida. O sistema de colisão com o terreno adapta-se a cada cenário para que o drone voe acima do solo e dos obstáculos.

Seleção de Idioma

O Drone Commander suporta vários idiomas. Use o menu no painel esquerdo para alternar entre:

  • English — Inglês
  • Italiano — Italiano
  • Français — Francês
  • Deutsch — Alemão
  • Español — Espanhol
  • Português — Português

A configuração de idioma é salva no seu navegador e persiste entre sessões. Alterar o idioma atualiza a caixa de ferramentas, os rótulos dos blocos e todo o texto da interface.

Programas de Exemplo

1. Descolagem e Aterragem Básicas

O programa mais simples — o drone sobe e volta a descer.

  Start
  ├── decolar
  └── aterrar
  End

O que acontece: O drone eleva-se a 10m, paira brevemente, depois desce e aterra no solo abaixo.

2. Voo para a Frente com Regresso

Voe para a frente, vire-se e volte.

  Start
  ├── decolar
  ├── andar 10
  ├── definir ângulo 180
  ├── andar 10
  └── aterrar
  End

O que acontece: O drone descola, voa 10 metros para a frente, vira 180°, voa 10 metros para trás e aterra no local original.

3. Padrão Quadrado

Voe num quadrado movendo-se e virando repetidamente.

  Start
  ├── decolar
  ├── definir altitude 5
  ├── repetir 4 vezes
  │   ├── andar 5
  │   └── alterar ângulo 90
  └── aterrar
  End

O que acontece: O drone voa num padrão quadrado de 5×5 metros a 5m de altitude antes de aterrar.

4. Drone Controlado por Teclado

Use o teclado para pilotar o drone em tempo real. Isto usa um laço while verdadeiro com blocos de sensores.

  Start
  ├── decolar
  ├── enquanto verdadeiro
  │   ├── se tecla "Seta para Cima" pressionada
  │   │   └── andar 0.5
  │   ├── se tecla "Seta para Baixo" pressionada
  │   │   └── andar -0.5
  │   ├── se tecla "Seta para Esquerda" pressionada
  │   │   └── alterar ângulo -5
  │   ├── se tecla "Seta para Direita" pressionada
  │   │   └── alterar ângulo 5
  │   └── esperar 0.1
  └── aterrar
  End
Dica: Ao executar programas controlados por teclado, clique primeiro na vista 3D para garantir o foco do teclado e, em seguida, use as setas para controlar o drone.

5. Voo com Sensores de Altitude

Suba a altitude, mova-se para a frente e use sensores.

  Start
  ├── decolar
  ├── definir altitude 8
  ├── definir velocidade 3
  ├── andar 15
  ├── alterar ângulo 90
  ├── fumaça verdadeiro
  ├── andar 10
  ├── fumaça falso
  └── aterrar
  End

O que acontece: O drone sobe a 8m, move-se para a frente rapidamente a velocidade 3, vira, deixa um rasto de fumo por 10m e depois aterra.

6. Patrulha Automática com Laços

Patrulhe uma área retangular continuamente (até ser parado).

  Start
  ├── decolar
  ├── definir altitude 4
  ├── definir velocidade 2
  ├── enquanto verdadeiro
  │   ├── andar 8
  │   ├── alterar ângulo 90
  │   ├── andar 4
  │   ├── alterar ângulo 90
  │   └── esperar 0.5
  └── aterrar
  End

O que acontece: O drone voa num padrão de patrulha retangular a 4m de altitude até clicar em Parar.

Dicas e Resolução de Problemas

Melhores práticas

  • Comece com Start: Inicie sempre o seu programa com um bloco Start para definir claramente o ponto de entrada da execução.
  • Descole primeiro: Os comandos de movimento (andar, andar subindo, deslizar, ir para, mover por, curva abs, curva, voltar à base, etc.) só funcionam enquanto o drone está em voo. Inclua sempre decolar antes do movimento.
  • Consciência do terreno: O drone adapta-se automaticamente à altura do terreno. Ao aterrar, desce ao nível do solo na sua posição atual, que pode variar em cenários montanhosos.
  • Controlo de velocidade: O bloco velocidade afeta a rapidez com que o drone se move com andar, andar subindo, ir para, mover por, curva abs, curva e voltar à base. O intervalo de velocidade é 0–10; o predefinido é 1.0.
  • Fumo como ferramenta de depuração: Ative o fumo para visualizar a trajetória de voo do drone.

Comportamentos conhecidos

  • Redefinição do drone: Quando executa um programa, o drone é automaticamente redefinido para a sua posição inicial (centro do mapa) antes da execução.
  • Deteção de colisões: O drone não atravessa o solo ou objetos do terreno. A altitude é automaticamente limitada para permanecer acima do terreno.
  • Salvamento automático: Funciona em todos os navegadores modernos. Limpar o armazenamento local do seu navegador irá eliminar os programas e preferências salvos.
  • Sem necessidade de internet: Após o carregamento inicial da página, o Drone Commander funciona inteiramente no seu navegador. Todas as bibliotecas são carregadas localmente.

Se algo não funcionar

  • O drone não se move: Certifique-se de que o drone descolou. O estado de voo deve mostrar "Em Voo".
  • O botão Executar permanece desativado: Clique em Parar, depois clique em Parar novamente para redefinir. O botão Executar será reativado.
  • A vista 3D está em branco: Verifique se o seu navegador suporta WebGL. Tente um navegador diferente (recomendados Chrome, Firefox, Edge).
  • Sem som: O som do drone requer interação do utilizador (clique) para iniciar — isto é uma política de reprodução automática do navegador. Clique no botão Executar; se o áudio ainda não tocar, verifique as configurações de som do seu navegador para o separador.
  • Desempenho lento: Mude para o perfil gráfico Desempenho no menu.

Detalhes Técnicos

  • Motor: Three.js (renderização 3D), Blockly (programação visual)
  • Formato: Ficheiro único index.html com JavaScript incorporado
  • Armazenamento: Armazenamento local do navegador para salvamento automático, preferências e seleção de idioma
  • Modelos: STL (drone, hélices, câmara), OBJ/MTL (objetos de cenário)
  • Som: Web Audio API com drone.mp3 em loop durante o voo
  • Física do drone: Fila de comandos com interpolação suave (sem motor físico real; os movimentos são animados)
