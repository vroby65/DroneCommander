Drone Commander

Drone Commander is an interactive web application that lets you simulate drone flight control using visual programming. You build a flight program by dragging and dropping Blockly blocks, then watch the drone execute those commands in a real-time 3D environment powered by Three.js.

No coding experience is required — just snap blocks together and press Run!

Quick start: Drag a Start block, attach a take off block, then a land block, and click ▶ Run. Your drone will take off and land!

Interface Overview

The application is divided into two main panels separated by a resizable divider:

  • Left Panel — The Blockly editor where you build your flight program.
  • Right Panel — The 3D simulation view with a status bar at the bottom.

Left panel menu bar (top of the editor):

  • Language selector — Switch the interface language.
  • New — Create a new empty program.
  • Load — Load a Blockly program from an .xml file on your computer.
  • Save — Save your current program as an .xml file.
  • Scenario dropdown — Choose a 3D environment (see Scenarios).
  • Graphics dropdown — Choose a graphics quality preset (see Graphics Profiles).

Right panel toolbar (above the 3D view):

  • 🗖 / 🗗 — Toggle the 3D view to fullscreen (hides the editor and status panel).
  • File name — The name of the current program.
  • ▶ Run — Execute the Blockly program.
  • ■ Stop — Stop the simulation; click again to reset the drone.

The Blockly Editor

The Blockly editor is a visual programming environment. You create a program by dragging blocks from the toolbox (the sidebar on the left of the editor) onto the workspace (the large white area), and snapping them together like puzzle pieces.

How to build a program

  1. Every program should start with a Start block (from the Flow category).
  2. Attach Drone command blocks below the Start block in the order you want them to execute.
  3. Optionally end with an End block.
  4. Click ▶ Run to see the drone execute your program.

Common editing actions

  • Drag — Move a block from the toolbox or around the workspace.
  • Right-click a block for context options: Duplicate, Delete, Add Comment, Disable, Inline Inputs, etc.
  • Numeric inputs — Click on a number field and type a value. You can also attach a Math block (e.g., a number block or arithmetic block) to input slots.
  • Boolean inputs (e.g., the smoke block) accept True/False blocks, logic blocks, or sensor blocks.
  • Disable a block — Right-click > Disable Block to temporarily remove it from the program without deleting it.
  • Collapse — Right-click > Collapse Block to reduce visual clutter.

Block Reference

The toolbox on the left side of the editor is organized into the following categories:

Logic Loops Math Text Lists Variables Functions Sensors Drone Flow

Logic

Standard if/else conditions, logical operators (and, or, not), boolean values (true/false), comparison operators (=, ≠, <, ≤, >, ≥), and the ternary operator.

Loops

Repeat blocks a fixed number of times, while/until loops, counting loops (for), for-each loops, and loop control statements (break, continue).

Math

Number constants, arithmetic operations (+, −, ×, ÷, ^), square root, absolute value, trigonometric functions, rounding, random numbers, and number property checks (even, odd, prime, etc.).

Text

String constants, text concatenation, length, substring, character lookup, case conversion, trimming, and printing to the console.

Lists

Create lists, get/set items, find sublists, sort, reverse, check emptiness, and split/join strings.

Variables

Create and use variables to store and retrieve values during program execution.

Functions

Define reusable procedures (with or without return values) to organize your code.

Drone Blocks

These are the core blocks for controlling the drone. Blocks marked flying required will only take effect while the drone is airborne; they are silently skipped if the drone is on the ground.

Block Description Requires Flying
take off Starts the drone's propellers and lifts it to a height of 10 meters. The drone enters flying state. No
land Descends the drone to the ground, stops the propellers, and exits flying state. The drone lands at the terrain height below it. Yes
set altitude [N] Change the drone's altitude to the specified value (in meters). The drone smoothly transitions to the target altitude. Yes
change altitude [N] Adjust the drone's altitude by a relative amount (e.g., +5 to go up 5m, -3 to go down 3m). Yes
set angle [N] Rotate the drone to face a specific direction (0–359 degrees, where 0 = north). Yes
change angle [N] Rotate the drone by a relative number of degrees (e.g., 90 for a right turn, -90 for left). Yes
slide [N] Move the drone sideways (left/right) by the specified distance. Positive values move right, negative move left. Yes
walk [N] Move the drone forward by the specified distance (in meters). The drone moves in the direction it is currently facing. Yes
go to x [X] y [Y] z [Z] Move the drone to an absolute position, where Y is altitude. The drone turns and leans toward the destination and uses the current speed setting. Yes
move by x [X] y [Y] z [Z] Move the drone by a relative offset from its current position. The drone turns and leans toward the destination and uses the current speed setting. Yes
return to base Move the drone back to the base at X=0, Z=0 and altitude 10. The drone turns and leans toward the base and uses the current speed setting. Yes
wait [N] Pause the program for the specified number of seconds before continuing to the next block. No
smoke [True/False] Enable or disable the smoke trail effect. When enabled, the drone leaves a visible trail as it flies. No
speed [N] Set the drone's movement speed (0–10). Higher values make the drone move faster during walk, go to, move by, and return to base commands. The default speed is 1.0. No
Note: The drone can only walk, slide, go to, move by, and return to base while flying. set_angle and change_angle change the drone's facing direction without moving it.

Sensor Blocks

Sensor blocks return real-time data from the drone simulation. They can be plugged into number or boolean inputs of other blocks (e.g., inside conditions of if/while blocks).

Block Returns Description
key [KEY] pressed? Boolean Returns true if the specified keyboard key is currently pressed. Available keys: Arrow Up/Down/Left/Right, Space, Return, A–Z.
X Number Returns the drone's current X coordinate (east-west position).
Z Number Returns the drone's current Z coordinate (north-south position).
Altitude Number Returns the drone's current altitude in meters.
Direction Number Returns the drone's current heading direction in degrees (0–359).
speed Number Returns the drone's current speed setting (0–10).

Using sensors — suggested patterns

  • Keyboard driving: Use key pressed inside a while true loop with if blocks to make the drone react to arrow keys in real time.
  • Conditional altitude: Use the Altitude sensor with a logic comparison to keep the drone above a certain height.
  • Auto-landing: Check if altitude is near 0 and then trigger a land block.

Flow Control Blocks

Block Description
Start The entry point of every program. All drone commands must be connected below this block. It has a "hat" shape and no connection on top.
End An optional block to mark the end of the program. Any blocks below it will not execute.
Tip: If you don't include a Start block, the program will still run but may execute blocks in unexpected order. Always start with Start.

Simulation Controls

Run (▶)

Click the green ▶ button in the right panel toolbar to execute your Blockly program. The program is converted to JavaScript and run step by step. While running, the Stop button is enabled and the Run button is disabled.

Stop (■)

Click the red ■ button to stop the running program. The drone stops immediately. Click Stop again to reset the drone to its starting position.

What happens during execution

  • Each block runs sequentially with a small delay between commands so you can see the movement.
  • The drone's sound (propeller noise) plays during flight.
  • The status panel updates in real time with position, altitude, and direction.
  • Propellers spin visibly during flight.
  • Smoke trails are rendered if smoke mode is enabled.

3D View & Camera Controls

The 3D view shows the drone flying in the selected scenario. You can control the camera freely:

Action Control
Orbit (rotate view) Right-click + drag to rotate the camera around the drone.
Reset view Middle-click (mouse wheel button) to snap the camera behind the drone.
Zoom Scroll wheel to zoom in and out.
First-person Left-click + drag while the drone is flying to switch to a chase-cam view that follows the drone closely.

Drone visual guides

When flying, two visual aids appear (in Balanced and Performance modes):

  • Ground marker — A blue ring on the ground directly below the drone.
  • Altitude line — A vertical blue line connecting the drone to its ground marker, showing the height.

These guides can be hidden by switching to Quality mode or setting a custom profile.

Status Panel

The status panel is shown at the bottom of the right panel. It displays the drone's current telemetry and allows manual adjustments:

Field Description
X The drone's east-west position. Editable — change the value and press Enter to teleport the drone (collision-checked).
Z The drone's north-south position. Editable like X.
Altitude The drone's height above ground. Editable — useful for quickly positioning the drone.
Direction The drone's heading in degrees (0–359). Editable — change to rotate the drone instantly.
Status Shows "On Ground" or "In Flight" depending on the drone's state.

File Management

New

Click New to clear the current program and start with a fresh workspace containing just a Start block. You'll be prompted if you want to save the current program first.

Save

Click Save to download your current program as an .xml file. You will be prompted for a file name. The program is also saved to your browser's local storage automatically.

Load

Click Load to open an .xml file from your computer. The loaded program replaces the current workspace content.

Autosave

Your program is automatically saved to your browser's local storage every time you make a change. When you return to the page or refresh it, the last saved program is restored automatically. The current file name is also remembered across sessions.

Scenarios

Drone Commander includes four 3D environments that you can switch between using the dropdown in the left panel menu.

Scenario Description
Campo di volo (Flight Field) A grassy airfield with trees, benches, and moderate rolling hills. The default environment.
Pista urbana (Urban Track) A city environment with buildings and urban scenery.
Metropoli (Metropolis) A large city with skyscrapers and dense urban layout.
Isola tropicale (Tropical Island) A tropical island with water, sand, and island vegetation.

When you switch scenarios, the drone resets to the center of the new environment and the camera returns to its default position. The terrain collision system adapts to each scenario so the drone flies above the ground and obstacles.

Example Programs

1. Basic Takeoff and Landing

The simplest program — the drone goes up and comes back down.

  Start
  ├── take off
  └── land
  End

What happens: The drone lifts to 10m, hovers briefly, then descends and lands on the ground below.

2. Forward Flight with Return

Fly forward, turn around, and come back.

  Start
  ├── take off
  ├── walk 10
  ├── set angle 180
  ├── walk 10
  └── land
  End

What happens: The drone takes off, flies forward 10 meters, turns 180°, flies back 10 meters, and lands at the original spot.

3. Square Pattern

Fly in a square by repeatedly moving and turning.

  Start
  ├── take off
  ├── set altitude 5
  ├── repeat 4 times
  │   ├── walk 5
  │   └── change angle 90
  └── land
  End

What happens: The drone flies a 5×5 meter square pattern at 5m altitude before landing.

4. Keyboard-Controlled Drone

Use the keyboard to fly the drone in real time. This uses a while true loop with sensor blocks.

  Start
  ├── take off
  ├── while true
  │   ├── if key "Arrow Up" pressed
  │   │   └── walk 0.5
  │   ├── if key "Arrow Down" pressed
  │   │   └── walk -0.5
  │   ├── if key "Arrow Left" pressed
  │   │   └── change angle -5
  │   ├── if key "Arrow Right" pressed
  │   │   └── change angle 5
  │   └── wait 0.1
  └── land
  End
Tip: When running keyboard-controlled programs, click on the 3D view first to ensure keyboard focus, then use the arrow keys to control the drone.

5. Altitude-Aware Flight

Climb to altitude, move forward, and use sensors.

  Start
  ├── take off
  ├── set altitude 8
  ├── set speed 3
  ├── walk 15
  ├── change angle 90
  ├── smoke true
  ├── walk 10
  ├── smoke false
  └── land
  End

What happens: The drone climbs to 8m, moves forward quickly at speed 3, turns, leaves a smoke trail for 10m, then lands.

6. Automated Patrol with Loops

Patrol a rectangular area continuously (until stopped).

  Start
  ├── take off
  ├── set altitude 4
  ├── set speed 2
  ├── while true
  │   ├── walk 8
  │   ├── change angle 90
  │   ├── walk 4
  │   ├── change angle 90
  │   └── wait 0.5
  └── land
  End

What happens: The drone flies a rectangle patrol pattern at 4m altitude until you click Stop.

Tips & Troubleshooting

Best practices

  • Start with Start: Always begin your program with a Start block to define the execution entry point clearly.
  • Take off first: Movement commands (walk, slide, go to, move by, return to base, etc.) only work while the drone is in flight. Always include take off before movement.
  • Terrain awareness: The drone automatically adapts to terrain height. When landing, it descends to the ground level at its current position, which may vary in hilly scenarios.
  • Speed control: The speed block affects how fast the drone moves with walk, go to, move by, and return to base. Speed range is 0–10; default is 1.0.
  • Smoke as a debugging tool: Enable smoke to visualize the drone's flight path.

Known behaviors

  • Drone reset: When you run a program, the drone automatically resets to its starting position (center of the map) before execution.
  • Collision detection: The drone will not clip through the ground or terrain objects. Altitude is automatically clamped to stay above terrain.
  • Autosave: Works in all modern browsers. Clearing your browser's local storage will delete saved programs and preferences.
  • No internet required: After the initial page load, Drone Commander runs entirely in your browser. All libraries are loaded locally.

If something doesn't work

  • Drone doesn't move: Make sure the drone has taken off. Flight status should show "In Flight".
  • Run button stays disabled: Click Stop, then click Stop again to reset. The Run button will re-enable.
  • 3D view is blank: Check that your browser supports WebGL. Try a different browser (Chrome, Firefox, Edge recommended).
  • No sound: The drone sound requires user interaction (click) to start — this is a browser autoplay policy. Click the Run button; if audio still doesn't play, check your browser's sound settings for the tab.
  • Slow performance: Switch to the Performance graphics profile in the dropdown.

Technical Details

  • Engine: Three.js (3D rendering), Blockly (visual programming)
  • Format: Single index.html file with embedded JavaScript
  • Storage: Browser local storage for autosave, preferences, and language selection
  • Models: STL (drone, propellers, camera), OBJ/MTL (scenery objects)
  • Sound: Web Audio API with drone.mp3 looped during flight
  • Drone physics: Command queue with smooth interpolation (no real physics engine; movements are animated)
